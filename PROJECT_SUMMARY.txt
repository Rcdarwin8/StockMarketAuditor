🔧 TECHNICAL IMPLEMENTATION DETAILS
═══════════════════════════════════════════════════════════════════════════

1. XPATH STRATEGY
   ─────────────────
   Base Table:
     //table[contains(@class, 'mctable')]
   
   Row Indexing (skip header):
     //table[contains(@class, 'mctable')]//tr[i+1]
   
   Cell Navigation:
     rowXPath + "/td[1]//a"    → Company Name (inside <a> tag)
     rowXPath + "/td[3]"       → Previous Close
     rowXPath + "/td[4]"       → Current Price
     rowXPath + "/td[5]"       → Website % Change


2. DATA CLEANING ALGORITHM
   ────────────────────────
   Input:  "₹2,450.75"
   Step 1: Remove currency symbols → "2,450.75"
   Step 2: Remove commas           → "2450.75"
   Step 3: Trim whitespace         → "2450.75"
   Step 4: Parse to Double         → 2450.75

   Regex Pattern: .replaceAll("[₹$,\\s]", "")


3. CALCULATION FORMULA
   ────────────────────
   percentageChange = ((currentPrice - previousClose) / previousClose) × 100
   
   Example:
     Previous: 2450.00
     Current:  2580.00
     Change:   ((2580 - 2450) / 2450) × 100 = 5.31%


4. VALIDATION LOGIC
   ─────────────────
   tolerance = 0.1  // 0.1% tolerance for rounding
   difference = |calculatedPercent - websitePercent|
   
   if (difference ≤ tolerance):
       status = "✓ MATCH"
       validationPassed = true
   else:
       status = "✗ MISMATCH"
       validationPassed = false


5. EXCEL STYLING CODE
   ───────────────────
   // Currency format
   style.setDataFormat(workbook.createDataFormat().getFormat("₹#,##0.00"));
   
   // Percentage format
   style.setDataFormat(workbook.createDataFormat().getFormat("0.00\"%\""));
   
   // Validation PASS style
   font.setColor(IndexedColors.DARK_GREEN.getIndex());
   style.setFillForegroundColor(IndexedColors.LIGHT_GREEN.getIndex());
   
   // Validation FAIL style
   font.setColor(IndexedColors.DARK_RED.getIndex());
   style.setFillForegroundColor(IndexedColors.ROSE.getIndex());

🐛 TROUBLESHOOTING GUIDE
═══════════════════════════════════════════════════════════════════════════

Problem: "ChromeDriver not found"
Solution:
  mvn clean install -U
  (WebDriverManager will auto-download ChromeDriver)

Problem: "Table not loading / NoSuchElementException"
Solution:
  • Check internet connection
  • Verify website is accessible in regular browser
  • Increase wait time: Duration.ofSeconds(30)
  • Website structure may have changed (update XPath)

Problem: "NumberFormatException during parsing"
Solution:
  • Add debug logging: System.out.println("Raw text: " + priceText);
  • Check for new special characters in prices
  • Update cleanAndParsePrice() regex pattern

Problem: "Excel file not generated"
Solution:
  • Check output directory exists: mkdir -p /mnt/user-data/outputs
  • Verify write permissions: chmod 755 /mnt/user-data/outputs
  • Check disk space: df -h

Problem: "No high performers found"
Solution:
  • Market may be down (all stocks declining)
  • Lower threshold: HIGH_PERFORMANCE_THRESHOLD = 3.0
  • Check different time of day (market hours)

Problem: "Validation failures"
Solution:
  • Website may round differently
  • Increase tolerance: double tolerance = 0.2
  • This is expected behavior, not a bug
